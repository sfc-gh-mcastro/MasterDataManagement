### Implementation steps

#### Step 1 ‚Äî DT 1: `CRMI_AGG_DT_ENRICHED`

**Purpose**: Normalize schemas, validate personnummer, resolve Norwegian nicknames.

Key CTEs:
1. `nicknames` ‚Äî inline `VALUES` table of ~18 Norwegian nickname ‚Üí canonical pairs (e.g., `PETTER‚ÜíPER`, `OLAV‚ÜíOLE`, `ERIK‚ÜíEIRIK`, `ELISABETH‚ÜíLISE`, `KARIN‚ÜíKARI`, `MARIA‚ÜíMARIE`, etc.)
2. `raw_freg / raw_bs / raw_nice` ‚Äî add `SOURCE_SYSTEM`, `TRUST_SCORE` (3/2/1), pad missing columns to NULL with explicit casts
3. `unioned` ‚Äî `UNION ALL` of the three
4. `ssn_digits` ‚Äî extract `d1..d11` as integers only where `SSN RLIKE '^[0-9]{11}$'`
5. `ssn_weighted` ‚Äî compute `r1 = MOD(3d1+7d2+6d3+d4+8d5+9d6+4d7+5d8+2d9, 11)` and `r2 = MOD(5d1+4d2+3d3+2d4+7d5+6d6+5d7+4d8+3d9+2d10, 11)` ‚Äî standard Norwegian modulus-11 weights
6. `ssn_validated` ‚Äî `SSN_VALID = TRUE` iff: 11-digit numeric, `r1 ‚âÝ 1`, `r2 ‚âÝ 1`, `(IF r1=0 THEN 0 ELSE 11-r1) = d10`, `(IF r2=0 THEN 0 ELSE 11-r2) = d11`
7. Final `SELECT` with `LEFT JOIN nicknames ON UPPER(FIRST_NAME) = nickname` ‚Üí `CANONICAL_FIRST_NAME`

Output adds: `SOURCE_SYSTEM`, `TRUST_SCORE`, `SSN_VALID`, `CANONICAL_FIRST_NAME`

---

#### Step 2 ‚Äî DT 2: `CRMI_AGG_DT_GROUPS`

**Purpose**: Entity resolution ‚Äî SSN exact match, Jaro-Winkler fuzzy fallback, steward queue flag.

Key CTEs:
1. `ssn_matched` ‚Äî where `SSN_VALID=TRUE`: `MATCH_ID=SSN`, `MATCH_CONFIDENCE=100`, `MATCH_METHOD='SSN_EXACT'`, `STEWARD_QUEUE_FLAG=FALSE`
2. `no_ssn_records` ‚Äî where `SSN IS NULL OR SSN_VALID=FALSE`
3. `fuzzy_candidates` ‚Äî `DISTINCT (SSN AS CANDIDATE_SSN, CANONICAL_FIRST_NAME, UPPER(LAST_NAME))` from valid-SSN records
4. `fuzzy_scores` ‚Äî `CROSS JOIN no_ssn √ó fuzzy_candidates`; compute `JW_FIRST = JAROWINKLER_SIMILARITY(canonical_first, cand_first)`, `JW_LAST = JAROWINKLER_SIMILARITY(upper_last, cand_last)`, `JW_AVG = (JW_FIRST + JW_LAST) / 2.0`; also carry `JW_AVG` as `BEST_JW_SCORE` for the steward queue
5. `best_fuzzy` ‚Äî `QUALIFY ROW_NUMBER() OVER (PARTITION BY row_key ORDER BY JW_AVG DESC) = 1` where `JW_AVG >= 92`
6. `fuzzy_matched` ‚Äî `MATCH_ID=CANDIDATE_SSN`, `MATCH_CONFIDENCE=85`, `MATCH_METHOD='FUZZY_JW'`, `STEWARD_QUEUE_FLAG=FALSE`
7. `best_attempt` ‚Äî best JW score below 92 for unmatched records (for steward queue context)
8. `unmatched` ‚Äî `no_ssn_records NOT IN fuzzy_matched` using `NOT EXISTS`; `MATCH_ID=NULL`, `MATCH_CONFIDENCE=0`, `MATCH_METHOD='UNMATCHED'`, `STEWARD_QUEUE_FLAG=TRUE`, carry `BEST_JW_SCORE` from `best_attempt`
9. Final `UNION ALL` of three groups

Output adds: `MATCH_ID`, `MATCH_CONFIDENCE`, `MATCH_METHOD`, `STEWARD_QUEUE_FLAG`, `BEST_JW_SCORE`

---

#### Step 3 ‚Äî DT 3: `CRMI_AGG_DT_GOLDEN`

**Purpose**: Org-partitioned survivorship ‚Äî one golden record per `(MATCH_ID, ORGANIZATION)`.

Key logic:
1. `org_combos` ‚Äî distinct `(MATCH_ID, ORGANIZATION)` from BS+NICE records; for FREG-only customers (no org) use `ORGANIZATION='FREG'`
2. `expanded` ‚Äî join `org_combos` to `CRMI_AGG_DT_GROUPS` on `MATCH_ID`, including FREG records for any org (`SOURCE_SYSTEM='FREG' OR g.ORGANIZATION=oc.ORGANIZATION`)
3. `survivorship` ‚Äî conditional aggregation per `(MATCH_ID, ORGANIZATION)`:
   - Name: `COALESCE(MAX(CASE WHEN src='FREG' THEN first_name END), MAX(CASE WHEN src='BS' THEN ‚Ä¶ END), MAX(CASE WHEN src='NICE' THEN ‚Ä¶ END))` using most-recent within tier: `MAX(CASE WHEN src='FREG' AND RECORD_DATE=(SELECT MAX from same partition) THEN ‚Ä¶`
   - Simpler: use `FIRST_VALUE(first_name) OVER (PARTITION BY match_id, org ORDER BY trust_score DESC, record_date DESC)` via `QUALIFY ROW_NUMBER()=1` after unnesting
   - `BIRTH_DATE`, `CITIZENSHIP`: FREG only
   - `PHONE`, `EMAIL`: BS (latest) then NICE (latest)
   - `SOURCE_COUNT`: `COUNT(DISTINCT SOURCE_SYSTEM)`
   - `MATCH_METHOD`: `MAX(MATCH_METHOD)` (propagate method)
4. `CUSTOMER_ID = MD5(MATCH_ID || '::' || ORGANIZATION)` ‚Äî stable synthetic key
5. `DQ_SCORE`:
   ```
   GREATEST(0, LEAST(100,
     100
     - IFF(PHONE IS NULL, 20, 0)
     - IFF(EMAIL IS NULL, 15, 0)
     - IFF(BIRTH_DATE IS NULL, 5, 0)
     - IFF(SOURCE_COUNT = 1, 10, 0)
     - IFF(MATCH_METHOD = 'FUZZY_JW', 20, 0)
   ))::SMALLINT
   ```

Output: `CUSTOMER_ID`, `MATCH_ID`, `ORGANIZATION`, `FIRST_NAME`, `LAST_NAME`, `BIRTH_DATE`, `CITIZENSHIP`, `PHONE`, `EMAIL`, `RECORD_DATE`, `SOURCE_COUNT`, `DQ_SCORE`, `MATCH_METHOD`

---

#### Step 4 ‚Äî DT 4: `CRMI_AGG_DT_CURRENT`

**Purpose**: One row per golden customer per org, with row hash for change detection.

Simple projection of `CRMI_AGG_DT_GOLDEN` plus:
- `ROW_HASH = SHA2(CONCAT_WS('::', FIRST_NAME, LAST_NAME, PHONE, EMAIL, COALESCE(BIRTH_DATE::VARCHAR,''), COALESCE(CITIZENSHIP,'')))`
- `REFRESHED_AT = CURRENT_TIMESTAMP()`

---

#### Step 5 ‚Äî DT 5: `CRMI_AGG_DT_HISTORY`

**Purpose**: SCD Type 2 ‚Äî all attribute versions over time, using append-only snapshots.

Since raw tables are append-only, each distinct `RECORD_DATE` per customer is a natural version boundary. Key CTEs:
1. `org_dates` ‚Äî `DISTINCT (MATCH_ID, ORGANIZATION, RECORD_DATE)` from `CRMI_AGG_DT_GROUPS` where `MATCH_ID IS NOT NULL`
2. `snapshot_attrs` ‚Äî for each `(MATCH_ID, ORG, RECORD_DATE)`, compute survivorship using only records **from that exact date** (FREG contributes across orgs, BS/NICE per-org):
   - `MAX(CASE WHEN src='FREG' AND record_date=snapshot_date THEN first_name END)` etc.
   - `COALESCE(freg_first, bs_first, nice_first)` for survivorship
3. `hashed` ‚Äî add `ROW_HASH = SHA2(CONCAT_WS('::', key attributes))`
4. `versioned` ‚Äî add `LAG(ROW_HASH) OVER (PARTITION BY MATCH_ID, ORGANIZATION ORDER BY SNAPSHOT_DATE)` ‚Üí keep only rows where hash changed (`QUALIFY LAG(ROW_HASH) OVER (...) IS DISTINCT FROM ROW_HASH OR LAG(ROW_HASH) OVER (...) IS NULL`)
5. Final ‚Äî add:
   - `VALID_FROM = SNAPSHOT_DATE`
   - `VALID_TO = COALESCE(LEAD(SNAPSHOT_DATE) OVER (...), '9999-12-31'::DATE)`
   - `IS_VALID = (VALID_TO = '9999-12-31'::DATE)`
   - `CUSTOMER_ID = MD5(MATCH_ID || '::' || ORGANIZATION)`

---

#### Step 6 ‚Äî DT 6: `CRMI_AGG_DT_STEWARD_QUEUE`

**Purpose**: Surface unmatched/invalid records for manual review.

From `CRMI_AGG_DT_GROUPS` where `STEWARD_QUEUE_FLAG = TRUE`, add:
- `QUEUE_REASON = CASE WHEN SSN IS NULL THEN 'NO_SSN' WHEN SSN_VALID=FALSE THEN 'INVALID_SSN_CHECKSUM' ELSE 'UNMATCHED' END`
- `BEST_JW_SCORE` (carried from groups layer)
- `REVIEWED = FALSE::BOOLEAN`
- `QUEUED_AT = CURRENT_TIMESTAMP()`
